# 第16章 Grafanaでのデータ活用 ― 仕様書

## 章の概要

**章の目的**: Grafanaで何ができるかを把握し、ダッシュボード作成やトレース検索をコーディングエージェントに具体的に指示できるようになる。PromQL／TraceQL／LogQLは「何ができるか」「どう指示するか」を重視する。

**目標ページ数**: 16p

**所属する部**: 第IV部 実践 ― 収集と可視化

## 前提知識

- 第5章（3シグナル）、第13章（アプリ側計装）

## 節の構成

### 16.1 データソースの概念

**概要**: Grafanaがデータを保持せず、バックエンドにクエリを投げるだけである点を改めて示す。

**図表**:
- 図16.1: Grafanaとデータソース（Prometheus／Tempo／Loki）の関係

**キーポイント**:
- データはバックエンド側にある
- データソース設定はクラスタに既存（本書では設定済み前提）

### 16.2 Prometheusデータソースで何ができるか

**概要**: メトリクスの時系列可視化、集計、アラートを可能性リストで示す。PromQLは代表的な逆引きパターンのみ。

**図表**:

**キーポイント**:

## 章末の理解度チェック問題

### 問題の方針
- 問題数: 4問
- 種類: 概念の確認1問、判断問題1問、設計問題2問

### 問題案
1. Grafanaとデータストアの関係を1文で述べよ。（概念の確認）
2. 「過去1時間のエラー率」を見たい。どのデータソースを使うか。（判断問題）
3. `travel-helper` のLLM呼び出しレイテンシをp50／p95／p99で見るパネルを設計せよ（データソース／クエリ要件／可視化タイプ）。（設計問題）
4. 「直近30分、`service.name=travel-helper-ch13` でエラーステータスのトレースを一覧表示」をコーディングエージェントに依頼する指示文を書け。（設計問題）
