# 第17章 End-to-Endフロー ― メトリクス追加からダッシュボード表示まで ― 仕様書

## 章の概要

**章の目的**: 「新しいメトリクスが欲しい」となったときに、アプリ側の計装追加からダッシュボードへの反映までのフローを自力で回せるようになる。本書で学んだ要素を統合し、改善ループの起点を作る。

**目標ページ数**: 14p

**所属する部**: 第IV部 実践 ― 収集と可視化

## 前提知識

- 第13章（手動計装）、第15章（Collector設定）、第16章（Grafana活用）

### 問題の方針
- 問題数: 4問
- 種類: 概念の確認1問、設計問題3問

### 問題案
1. E2E拡張フローの4ステップを挙げよ。（概念の確認）
2. 新しいメトリクス「ツール呼び出しの失敗率」を追加する計装からダッシュボードまでの設計を述べよ。（設計問題）
3. 新しいカスタムSpan「エージェントの判断理由」を追加する場合、アプリ側・Grafana側で何を行うか。（設計問題）
4. 本書の範囲で獲得した知識を活かし、読者自身のエージェントに最初に追加すべき観測項目を3つ挙げよ。（設計問題）
