# 章仕様書作成スキル

`docs/book-plan.md`に基づいて各章の仕様書を作成する。仕様書は執筆の設計図であり、ここで定義した構成に沿って章が執筆される。

## 前提条件

- `docs/book-plan.md`（書籍企画書）が存在すること

## 出力先

```
docs/chapter-specs/chNN-spec.md   # NNは2桁ゼロ埋め（01〜）
```

## テンプレート

章仕様書のテンプレート: `./template.md`

## 設計の原則

### 章間の接続を先に設計する

各章の「前の章からの接続」と「次の章への接続」を必ず定義する。読者が章をまたいで学習する際に文脈が断絶すると理解度が大幅に下がるため、仕様書の段階で接続ポイントを設計しておく。

### 図表を仕様書の段階で特定する

各節に必要な図表を仕様書作成時に特定する。これにより執筆時に「図表ファースト」の原則（図表を先に設計し、本文は図の解説として書く）を守りやすくなる。

### 理解度チェック問題の方針を定める

章末に3〜5問の確認問題を設ける。以下の種類を混在させる:
- **概念の確認**: 用語の定義、パターンの特徴
- **判断問題**: ユースケースに対するパターン選択
- **設計問題**: 与えられた要件に対するアーキテクチャ設計

### サンプルコード要件（該当する場合）

`docs/sample-code-spec/`が存在する場合、その章で実装すべきサンプルコードの機能を「サンプルコード要件」セクションとして仕様書に含める。
