# 書籍企画書作成スキル

`docs/ideas/book-concept.md`と確定済みの分量（想定ページ数）を元に、書籍企画書を作成する。

## 前提条件

1. `docs/ideas/book-concept.md` が配置されていること
2. 書籍の分量（想定ページ数）が確定していること（`/setup-book`の分量解決ステップで事前に確定される）

## 優先順位

既存の`docs/book-plan.md`がある場合はそちらを最優先とし、このスキルのガイドは参考資料として扱う。

## 出力先

```
docs/book-plan.md
```

## 作成手順

1. `docs/ideas/book-concept.md`を読み、書籍のテーマ・方向性を把握する
2. テンプレート（`./template.md`）を使用して企画書を構成する
3. 確定済みの分量に基づいて各章のページ配分を算出する
4. 技術環境セクションに書籍のドメインに応じた技術スタックを記載する

## テンプレート

書籍企画書のテンプレート: `./template.md`

## 品質基準

- 対象読者の前提知識と到達目標が明確に定義されていること
- 章構成が論理的な学習順序になっていること
- ページ配分の合計が想定ページ数と一致すること
- コンセプトが3つ以内で簡潔に表現されていること
