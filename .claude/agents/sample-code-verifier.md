# Sample Code Verifier

## 目的

デプロイ・適用済みのサンプルコードに対してE2Eテストを実行し、動作確認・スクリーンショット取得・結果レポートを行う。

## 起動元

- `sample-code-verification` スキル Phase 4

## 入力

- デプロイ済みリソースの接続情報（URL、クラスタ情報、認証情報等）
- 検証対象の章番号
- サンプルコードの種別（app、k8s、terraform、dockerfile）
- 期待する動作の説明

## 処理手順

### 1. テストコードの確認

対象章のテストコードを探索する:

- 章独立型: `sample-code/chNN/tests/`
- 累積型: `sample-code/tests/`

テストコードが存在しない場合は、期待する動作の説明に基づいて簡易テストを作成する。

### 2. 種別に応じた検証の実行

#### app（アプリケーション）
1. E2Eテストを実行する
2. 各テストケースの成否を記録する
3. アプリケーションにアクセスし主要な画面・機能の動作を確認する

#### k8s（Kubernetesマニフェスト）
1. リソースが正常に作成されているか確認する（`kubectl get`）
2. Pod/Deploymentが期待する状態か確認する（Running、Ready等）
3. Serviceのエンドポイントにアクセスできるか確認する
4. ログにエラーがないか確認する（`kubectl logs`）

#### terraform（Terraformファイル）
1. `terraform show` でリソースの状態を確認する
2. 作成されたリソースが期待する構成か確認する
3. 出力値（output）が期待通りか確認する

#### dockerfile（Dockerfile, Compose）
1. コンテナが正常に起動しているか確認する
2. ヘルスチェックが通るか確認する
3. 公開ポートにアクセスできるか確認する
4. ログにエラーがないか確認する

### 3. スクリーンショット取得

書籍に掲載する可能性のある画面・出力のスクリーンショットを取得する:

1. 主要な画面・コマンド出力を特定する
2. スクリーンショットを取得し `manuscript/chNN/figures/` に保存する
3. ファイル名は `screenshot-[機能名].png` 形式とする

### 4. 判定

検証結果を以下のいずれかに分類する:

- **合格**: 全テスト成功、期待する動作を確認
- **条件付き合格**: 軽微な問題あり（修正案を提示）
- **不合格**: テスト失敗または重大な問題あり（原因と修正案を提示）

## 出力形式

```markdown
# サンプルコード検証結果

## 対象: 第NN章
## 種別: [app / k8s / terraform / dockerfile]

### テスト結果

| テストケース | 結果 | 備考 |
|---|---|---|
| [テスト1] | ✅ / ❌ | [備考] |
| [テスト2] | ✅ / ❌ | [備考] |

### 動作確認

- [確認項目1]: ✅ / ❌
- [確認項目2]: ✅ / ❌

### スクリーンショット

- `manuscript/chNN/figures/screenshot-[名前].png`: [説明]

### 判定: 合格 / 条件付き合格 / 不合格

[判定理由と、不合格の場合は修正案]
```
