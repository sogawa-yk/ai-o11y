# Mermaid Syntax Checker

## 目的

原稿に含まれるMermaid記法の図表が構文エラーなくレンダリングできることを検証する。また、figure-designスキルで定義された配色ルール（テーマ設定・classDef）の適用状況も確認する。

## 起動元

- `/review-chapter` ステップ6
- `/write-all` サブステップB

## 入力

- 対象原稿: `manuscript/chNN/chNN.md`
- 参照: figure-designスキルの配色ルール

## 処理手順

### 1. Mermaidコードブロックの検出

原稿から ````mermaid` で始まるコードブロックを全て抽出する。存在しない場合は「Mermaidブロックなし」として報告する。

### 2. 構文チェック

各Mermaidブロックについて以下を検証する:

- **図の種類宣言**: `graph`, `flowchart`, `sequenceDiagram`, `stateDiagram-v2`, `classDiagram`, `erDiagram`, `gantt`, `pie` 等の有効な宣言で始まっているか
- **括弧・ブレースの対応**: `[]`, `()`, `{}`, `(())` 等の対応が取れているか
- **矢印構文**: `-->`, `-.->`, `==>`, `->>`, `-->>` 等が正しい形式か
- **ノードID**: 空白や特殊文字を含むIDが適切にクォートされているか
- **サブグラフ**: `subgraph`に対応する`end`があるか
- **日本語ラベル**: 日本語テキストが適切にクォートまたはブラケットで囲まれているか
- **init設定**: `%%{init: ...}%%` のJSON構文が正しいか（閉じ括弧、クォートの対応）
- **classDef**: classDef宣言で参照されるクラス名と、ノードで使用される `:::className` が一致するか
- **特殊文字のエスケープ**: ラベル内の `(`, `)`, `[`, `]` 等がエスケープされているか

### 3. 配色ルール準拠チェック

以下のルールへの準拠を確認する:

- `%%{init: ...}%%` テーマ設定が含まれているか
- graph/flowchart図に `classDef` が定義されているか
- sequenceDiagram に `themeVariables` が設定されているか

### 4. よくあるエラーパターンの検出

- ノードIDの重複定義
- 存在しないノードへの矢印
- `sequenceDiagram` でのアクター未定義参照
- `stateDiagram-v2` での不正な遷移構文
- `init` 設定のJSON内での末尾カンマ

## 出力形式

```markdown
# Mermaid構文検証結果

## 対象: 第NN章

### 検出されたMermaidブロック数: N個

### 各ブロックの検証結果

#### ブロック1: [図の種類] (行番号: L[開始行]-L[終了行])
- **構文チェック**: OK / エラー
  - [エラーの場合: 具体的な問題と修正案]
- **配色ルール**: 準拠 / 非準拠
  - [非準拠の場合: 不足している設定]
- **修正案**: [修正が必要な場合のコード]

...

### サマリー
- **構文エラー**: N件
- **配色ルール非準拠**: N件
- **修正が必要なブロック**: [ブロック番号のリスト]
```
