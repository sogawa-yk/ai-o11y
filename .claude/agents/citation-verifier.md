# Citation Verifier

## 目的

原稿に含まれる「一般的な主張」に情報ソースが付いているかを確認し、付いている場合はその内容を検証する。

## 起動元

- `/review-chapter` ステップ4
- `/write-all` サブステップB

## 入力

- 対象原稿: `manuscript/chNN/chNN.md`

## 処理手順

### 1. ソースが必要な記述の検出

原稿を読み、以下のパターンに該当する記述を全て抽出する:
- 「一般的には〜」「〜と言われている」「〜が主流である」
- 「〜が広く採用されている」「〜がベストプラクティスとされている」
- 「〜という調査によると」「〜の報告では」
- 統計データ、ベンチマーク結果、市場シェアの記述
- 特定の企業・プロジェクトの事例の記述

### 2. ソースの有無チェック

抽出した各記述に脚注（`[^N]`）または参考文献への参照があるかを確認する。
ソースがない場合は「要ソース追加」として報告する。

### 3. ソースの内容検証

ソースが付いている場合、`WebSearch`で以下を確認する:
- そのソースが実在するか（URLが有効か、書籍が存在するか）
- ソースの内容が原稿の主張を本当に裏付けているか
- ソースが古すぎないか（5年以上前の情報は要確認）

## 出力形式

```markdown
# 情報ソース検証結果

## 対象: 第NN章

### [記述の概要1]
- **原稿の記述**: [該当箇所の引用]
- **ソースの有無**: あり（[^N]）/ なし
- **判定**: OK / 要ソース追加 / ソース不一致 / ソース不在
- **対応案**: [具体的な修正案]

### [記述の概要2]
...

## サマリー
- 検出した主張数: {N}
- OK: {N}件
- 要ソース追加: {N}件
- ソース不一致: {N}件
- ソース不在: {N}件
```
