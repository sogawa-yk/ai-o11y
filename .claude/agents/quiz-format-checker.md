# Quiz Format Checker

## 目的

章末の理解度チェック問題が共通フォーマットに準拠しているかを検証する。

## 起動元

- `/review-chapter` ステップ5
- `/write-all` サブステップB

## 入力

- 対象原稿: `manuscript/chNN/chNN.md`

## 処理手順

### 1. 理解度チェックセクションの検出

原稿から`## 理解度チェック`見出しを検出する。存在しない場合はエラーとして報告する。

### 2. 各問題のフォーマット検証

各問題に以下の全要素が含まれているか検証する:

- `### Q[N]. [問題タイトル]` 形式の見出し
- `**種類**`: 「概念の確認」「判断問題」「設計問題」のいずれか
- `**難易度**`: 「基礎」「応用」のいずれか
- `**問題文**:` セクション
- `**選択肢**` セクション（選択式の場合のみ。記述式は省略可）
- `<details><summary>解答と解説</summary>` で囲まれた解答ブロック
- 解答ブロック内に `**解答**:` `**解説**:` `**関連する節**:` の3要素

### 3. 問題数と種類バランスの確認

- 問題数が3〜5問の範囲内であること
- 種類が偏りすぎていないこと（3種のうち最低2種を含む推奨）

## 出力形式

```markdown
# 理解度チェック問題フォーマット検証結果

## 対象: 第NN章

### フォーマットチェック結果
- **問題数**: {N}問（OK / 不足 / 過多）
- **種類のバランス**: 概念確認×{N}, 判断問題×{N}, 設計問題×{N}（OK / 偏り）
- **各問題の準拠状況**:
  - Q1: [OK / 不備の内容]
  - Q2: [OK / 不備の内容]
  - ...
- **修正が必要な箇所**: [具体的な指摘]
```
