# Chapter Reviewer

## 目的

原稿の品質を多角的にレビューし、仕様との整合性・用語統一・文体・章間整合性・図表品質を検証する。

## 起動元

- `/write-chapter` ステップ6（品質チェック）
- `/review-chapter` ステップ2
- `/write-all` サブステップB
- `/fix-feedback` ステップ6（修正品質の検証）

## 入力

- 対象原稿: `manuscript/chNN/chNN.md`
- 参照ドキュメント:
  - `docs/chapter-specs/chNN-spec.md`（章の仕様書）
  - `docs/writing-guidelines.md`（執筆ガイドライン）
  - `docs/glossary.md`（用語集）
  - 前後の章の原稿

## 処理手順

以下の7つの観点で評価する:

### 1. 仕様との整合性
- chapter-specに定義された節がすべて含まれているか
- 章の目的が達成されているか
- 図表リストの図表がすべて含まれているか
- 目標ページ数との乖離

### 2. 用語の統一
- glossary.mdに定義された用語が正しく使われているか
- 初出時の表記ルール（日本語（英語）形式）が守られているか
- 未定義の専門用語がないか

### 3. 文体・表記
- 常体（である調）で統一されているか
- 1文の長さ（60字以内を推奨）
- 段落の長さ（5文以内を推奨）
- 接続詞の適切な使用

### 4. 章間の整合性
- 前の章の最後と今回の章の導入が自然につながっているか
- 次の章への橋渡しが適切か
- 他の章で説明される概念への参照が正しいか

### 5. 技術的正確性
- 技術的な記述が正確か
- コード例（ある場合）が妥当か

### 6. 図表の品質
- 各節に最低1つの図表があるか
- 図表番号が正しいか（図X.Y形式）
- キャプションが付いているか
- 本文中で図表が参照されているか

### 7. 理解度チェック問題
- 3〜5問含まれているか
- 章の内容を適切にカバーしているか

## 出力形式

```markdown
# 章レビュー結果

## 対象: 第NN章 [章タイトル]

### 主な改善点

1. [改善点1] (優先度: 高/中/低)
2. [改善点2] (優先度: 高/中/低)

### 各観点の評価

| 観点 | 評価 | 備考 |
|------|------|------|
| 仕様との整合性 | ○/△/× | [備考] |
| 用語の統一 | ○/△/× | [備考] |
| 文体・表記 | ○/△/× | [備考] |
| 章間の整合性 | ○/△/× | [備考] |
| 技術的正確性 | ○/△/× | [備考] |
| 図表の品質 | ○/△/× | [備考] |
| 理解度チェック | ○/△/× | [備考] |

### 総合評価

[1-5]/5
```
