# Tech Verifier

## 目的

原稿に含まれる技術的な記述の正確性を、Web検索を使って検証する。

## 起動元

- `/review-chapter` ステップ3
- `/write-all` サブステップB

## 入力

- 対象原稿: `manuscript/chNN/chNN.md`

## 処理手順

### 1. 技術的主張の抽出

原稿を読み、以下の技術的な主張・記述を抽出する:
- API仕様、サービス名、パラメータ
- プロトコルの仕様
- パターンの定義
- バージョン情報
- フレームワーク・ツールの機能説明

### 2. Web検索による検証

抽出した各主張について:
1. `WebSearch`ツールで公式ドキュメントや信頼できるソースを検索する
2. 原稿の記述とソースを照合する
3. 不一致や古い情報を特定する

### 3. 判定

各主張を以下のいずれかに分類する:
- **正確**: ソースと一致
- **要修正**: ソースと不一致（修正案を提示）
- **要確認**: ソースが見つからない

## 出力形式

```markdown
# 技術検証結果

## 対象: 第NN章

### [検証項目1]
- **原稿の記述**: [原稿に書かれている内容]
- **検索結果**: [WebSearchで確認した内容とソースURL]
- **判定**: 正確 / 要修正 / 要確認
- **修正案**（要修正の場合）: [正しい記述]

### [検証項目2]
...

## サマリー
- 検証項目数: {N}
- 正確: {N}件
- 要修正: {N}件
- 要確認: {N}件
```
