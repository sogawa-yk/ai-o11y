# サンプルコードの再検証

**重要:** このコマンドは、既に `/write-chapter` で構築・検証済みのサンプルコードを再デプロイ・再検証するためのものです。初回の構築は `/write-chapter --env` で行ってください。

**引数:** 章番号 (例: `/deploy-sample-code 4`)

**必須引数:** `--env [環境名]` で検証環境を指定 (例: `/deploy-sample-code 4 --env local`)

---

## ステップ1: 前提確認

1. `--env` 引数が指定されていない場合、ユーザーに確認する
2. `docs/sample-code-spec/` の永続ドキュメントが存在することを確認する
3. サンプルコード（`sample-code/`）が存在することを確認する
   - 存在しない場合: 「サンプルコードが未構築です。`/write-chapter [章番号] --env [環境名]` で章の執筆と合わせて構築してください。」と案内して終了

## ステップ2: デプロイ

1. `Skill('sample-code-verification')` の Phase 4（デプロイ・検証）を実行する
2. 章独立型の場合: `sample-code/chNN/` をデプロイ
3. 累積型の場合: ブランチ `sample-code/chNN` のコードをデプロイ

## ステップ3: 検証

1. `Task`ツールを使用し、`sample-code-verifier` サブエージェントを起動して検証する
2. 検証結果を報告する

## ステップ4: クリーンアップ

1. 検証環境のリソースを削除する
2. クリーンアップの完了を確認する

## 完了条件

- サンプルコードの再検証が完了していること
- 検証結果がユーザーに報告されていること
- 環境のクリーンアップが完了していること
